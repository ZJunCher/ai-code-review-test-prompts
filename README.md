# AI Code Review Test Prompts

这个仓库整理了一组面向 Cursor 等 AI Coding 工具的项目规则提示词，主要用于作为 Project Rules 使用。将对应的 Markdown 文件添加到项目规则中后，AI 可以在代码审查、单元测试审查和测试覆盖率分析时参考这些规则，输出更稳定、结构化、可复用的审查建议。

## 内容

- `ai-code-review-prompt.md`  
  用于审查项目代码，关注代码结构、可读性、潜在 bug、异常处理、边界情况和可维护性。

- `unit-test-prompt.md`  
  用于审查或补充单元测试，帮助发现测试遗漏、断言不足、边界场景缺失等问题。

- `unit-test-coverage-prompt.md`  
  用于分析单元测试覆盖率，辅助判断哪些模块、分支或异常路径还需要补充测试。

## 使用方式

以 Cursor 为例，将需要使用的 Markdown 文件添加到项目的规则配置中，然后在对应项目中让 AI 执行代码审查、单元测试审查或覆盖率分析任务。

建议按以下顺序使用：

1. 先用代码审查规则检查项目代码；
2. 再用单元测试规则检查测试用例；
3. 最后用覆盖率规则分析测试覆盖情况。

## 适用场景

- AI Coding 工具项目规则配置
- 项目代码 Review
- 单元测试用例审查
- 测试覆盖率分析
- AI 辅助开发质量检查
